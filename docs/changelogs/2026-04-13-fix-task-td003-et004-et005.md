## TD_003_WorkOrderColumn — Persistent Failure (FIXED)

### Root Cause: Column name mismatch — "Name" not "Title"

The tasks grid's first column is labeled **"Name"**, not **"Title"**. This was confirmed by comparing two tests in the same file:

| Test | Priority | Checks for | Result |
|------|----------|------------|--------|
| TL_007_GridColumnHeaders | 7 | `"Name"` | PASSES |
| TD_003_WorkOrderColumn | 82 | `"Title"` | ALWAYS FAILS |

Both tests look at the same grid. TL_007 was written with the correct column names. TD_003 was added later with an incorrect assumption.

### Why all 3 fallback strategies failed

The `waitForColumnHeaderText()` function had `return result.contains("Title") ? result : null` as its success condition. Since "Title" never appears in column headers, the function **always timed out** and returned null — regardless of whether the grid was loaded.

Flow:
1. JS extraction → found headers but none contained "Title" → fallback
2. `waitForColumnHeaderText(15)` → polled for 15s looking for "Title" → always null
3. Page reload + retry → same result because "Title" still doesn't exist

### Fix

1. Changed all `"Title"` references to `"Name"` in TD_003 assertions
2. Changed `waitForColumnHeaderText()` success condition from `"Title"` to `"Name"`
3. Added explicit `driver.get(TASKS_URL)` at start of TD_003 (previous test may have navigated away)
4. Fixed `waitForGrid()` — added `waitAndDismissAppAlert()` after `driver.navigate().refresh()`

## ET_004_EditTaskDescription — Flaky (HARDENED)

**Error:** "Detail page should show Status field expected [true] but found [false]"

**Root cause:** The status check only looked for 4 values: "Pending", "Completed", "Scheduled", "Status". But the app uses 10+ status values including "Open", "In Progress", "Done", "To Do", "New", "Closed", "Cancelled". If the first task happened to have one of the uncovered statuses, the test failed.

**Fix:**
- Expanded status value check to include all known variants (case-insensitive)
- Added `dismissBackdrops()` before finding grid rows
- Added grid recovery if rows are empty
- Added `waitAndDismissAppAlert()` after `navigate().back()`
