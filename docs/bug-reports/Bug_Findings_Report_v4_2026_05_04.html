BUG-01 HIGH

Kebab menu does not close on ESC, outside-click, or toggle

Surface/assets/ (asset detail page)
Caught byTC_BH_13_KebabMenuClosesOnOutsideClick
EvidenceVisual screenshot + automated test

Description

The asset detail page kebab menu cannot be dismissed by ANY standard close mechanism. Focus trap. Violates WAI-ARIA menu pattern.

Steps to reproduce

  1. Navigate to /assets and log in.
  2. Click any row in the assets grid to open detail page.
  3. Click the three-dot kebab icon (top-right of detail header).
  4. Verify menu opens with "Edit Asset" / "Delete Asset" items.
  5. Press ESC key — menu stays open.
  6. Click on empty area of the page — menu stays open.
  7. Click kebab icon again (toggle attempt) — menu stays open.

Expected behavior

ESC + outside-click + toggle should each close the menu.

Actual behavior

Menu remains open after ALL three close methods.

Impact

Severe accessibility violation. Keyboard / screen-reader users blocked.

After pressing ESC and clicking outside, the kebab menu (Edit Asset / Delete Asset) is still visible top-right.

BUG-04 HIGH

Auth cookies use SameSite=None (CSRF risk) [SECURITY]

SurfaceAuthentication — Set-Cookie headers
Caught bySecurityAuditTestNG.testBUG007_AuthCookiesSameSiteNone
EvidenceHTTP response header inspection

Description

Authentication cookies are set with SameSite=None, allowing them to be sent on cross-site requests. Opens CSRF attack vector.

Steps to reproduce

  1. Open DevTools → Network tab.
  2. Log in to https://acme.qa.egalvanic.ai
  3. Inspect response headers of the login response.
  4. Check the SameSite attribute on Set-Cookie.

Expected behavior

SameSite=Lax (modern web default) or SameSite=Strict.

Actual behavior

SameSite=None — allows cross-site cookie attachment.

Impact

CSRF vulnerability. Attacker could forge requests on behalf of authenticated user. Block-on-ship.

No screenshot embedded — bug is observable directly via DevTools (Console / Network tab).
